# 时间同步
## 润秒
闰秒是偶尔运用于协调世界时（UTC）的调整，经由增加或减少一秒，以消弥精确的时间（使用原子钟测量）和不精确的观测太阳时（称为UT1），之间的差异。

## NTP (Network Time Protocol)
数据网络潜伏时间可变的计算机系统之间通过分组交换进行时钟同步的一个网络协议，位于OSI模型的应用层。自1985年以来，NTP是目前仍在使用的最古老的互联网协议之一。NTP通常可以在公共互联网保持几十毫秒的误差，并且在理想的局域网环境中可以实现超过1毫秒的精度。

NTP 就像你偶尔看一眼手机时间，把自己的表调一下。它通常用于服务器、办公网、互联网时间同步。优点是简单、通用、部署容易；缺点是精度一般，受操作系统调度、网络抖动影响比较大。

## PTP (Precision Time Protocol)
是一种可以将整个计算机网络时钟同步的网络传输协议,局域网通过该协议可以让时间精确到亚微秒级。它更像车队里选一个“标准表”，其他设备频繁和它对表，并且尽量把网络传输延迟也扣掉。Linux 下常用 linuxptp，它是 Linux 上的 IEEE 1588 PTP 用户态实现，并支持硬件和软件时间戳。可以结合网卡硬件时间戳，数据包刚进/出网卡那一刻就打时间。
PPS 是“很准的秒边沿”
PTP 是“通过网络把时间同步给其他设备”

## 术语
phc2sys:
PTP Hardware Clock
ptp4l:
PTP for Linux
PPS:
Pulse Per Second, 是“秒边沿脉冲”，不是完整时间,物理电平脉冲。通常每秒来一次。

## 已有demo梳理

    Orin 系统时间 CLOCK_REALTIME
            │
            │ phc2sys
            ▼
    Orin 网卡硬件时钟 /dev/ptp0，也叫 PHC
            │
            │ ptp4l
            ▼
    PandarXT 内部时钟
            │
            ▼
    点云包里的 UTC timestamp

``` bash
phc2sys 负责：把 Orin 系统时间喂给网卡硬件时钟 /dev/ptp0
ptp4l   负责：让 Orin 网卡通过 PTP 协议给雷达授时
雷达     负责：作为 PTP slave 跟着 Orin master 调自己的内部时间
```
